Welcome to the ESP8266Base project!

This project provides a generic framework and foundation for rapid IoT appliances creation.

The following features are available without writing a line of code:

  • Out-of-box network provisioning:
    • Automatically enter provisioning mode out-of-box;
    • Supports provisioning using the ESP SoftAP Provisioning app;
    • Also offers a built-in Web page (automatically prompted using a captive portal);
    • If the configured AP is unavailable at boot, will automatically enter provisioning mode;
      And if the AP recovers later, will automatically exit provisioning mode;
    • Provisioned AP config stored as a part of config file;
  • During regular servicing (after connected to an AP):
    • Automatic reconnect if connection to AP drops out
      For permanent AP turn-off, a reboot is needed to enter provisioning mode;
    • NTP time-sync with smooth time adjustment for small delta (<10 minutes);
    • Time tracking with RTC memory, and automatic restore at (powered-through) reboots;
    • Regular HTTP file serving with MIME type auto-inference for a number of built-in file extensions;
      Supports browser caching and ETag validation;
    • WebDAV for interactive Web application developing;
      (Can be disabled for production builds.)
  • Web-based appliance management:
    • Firmware image update and toggle.
    • User data partition backup, reset and restore.

The following functions are offered for programmatic integration:

  • Separate read-only system partition and read-write user partition:
    • The system partition is (usually) mounted read-only for safer operation;
    • Enabling WebDAV will mount the system partition read-write;
  • Extensible appliance configuration management API:
    • Parsing, validation, concurrency-safe access and modification;
    • Layering and delta storage of post-deployment changes (allows simple "reset-to-factory");
  • Configuration Web page sets:
    • Light & dark themed, mobile responsive;
    • Just plug in your app-specific configuration interface;
  • Structured RTC memory allocation and access:
    • RTC memory can be allocated for typed data structures;
      (Access should be 4-byte aligned.)
  • High-level system events and states:
    • When connected to configured AP (entering regular servicing);
    • When received IP address assignment from AP;
    • When the AP is disconnected and being retried;
    • When NTP time is received the first time after boot;
    • When some appliance component encountered unexpected failure;
